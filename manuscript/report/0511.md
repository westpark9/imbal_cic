### 2.2 FAR-MoE의 Vision Benchmark 확장 (2026-04-28, commit `d7af2c2`)

`code_mat.py`(CIC-IDS용 FAR-MoE)의 구조를 네 가지 CV 벤치마크로 포팅했다.

#### 추가된 스크립트 (`src/`)

| 스크립트 | 데이터셋 | Backbone | 특징 |
|---|---|---|---|
| `code_mat_cifar.py` | CIFAR-10/100-LT (IR=100) | ResNet-32 (scratch) | CIFAR 전용 32×32 아키텍처, cosine LR |
| `code_mat_imagenet.py` | ImageNet-LT (IR≈256, 1000 클래스) | ResNet-50 (pretrained fine-tune) | OLTR 어노테이션 자동 다운로드, tar 자동 추출 |
| `code_mat_inat.py` | iNaturalist-2018 (8,142 클래스) | ResNet-50 (pretrained fine-tune) | torchvision 자동 다운로드, `--max_classes` 서브셋 옵션 |
| `code_mat_mini_imagenet.py` | Downsampled ImageNet 64×64 | ResNet-50 (pretrained fine-tune) | pickle 배치 직접 로드, `--lt_sampling` 지수 LT 서브샘플링 |

#### 공통 파이프라인 구조

```
[Phase 1 — Backbone]
  ResNet 훈련 (CE loss, LT 데이터) → 특징 추출기

[Phase 2 — Probe]
  Backbone features → XGBoost probe → val 예측 → 정규화 혼동 행렬

[Phase 3 — Expert 설계]
  off-diagonal conf[i,j] > τ 쌍 → union-find 클러스터링
  전문가 로스터:
    Anchor       : 전 클래스, 균일 가중 (항상 포함)
    ConfMat_k    : 혼동 클러스터 k 내 클래스 upweight (OOD-게이팅)
    Tail         : threshold 미만 희귀 클래스 upweight (OOD-게이팅)
    ColSplit_0/1 : 특징 차원 반반 (OOD-게이팅)

[Phase 4 — Router]
  각 전문가 → LogisticRegression 역량 모델 (val에서 정답 여부 레이블)
  OOD 게이팅: LedoitWolf 공분산 → Mahalanobis z-score

[추론]
  final_proba = Σ_e α_e(x) · proba_e(x), argmax → 예측 클래스
```

#### CIC-IDS 대비 핵심 차이점

| 항목 | CIC-IDS (`code_mat.py`) | Vision 버전 |
|---|---|---|
| 특징 | 탭형 네트워크 통계량 | ResNet 심층 특징 (64 / 2048 dim) |
| 전문가 입력 | 원본 tabular feature | Backbone 출력 embedding |
| 전문가 종류 | Family/ConfMat/Tail/Random/Anchor | ColSplit/ConfMat/Tail/Anchor |
| Backbone | 없음 (XGBoost 직접) | ResNet-32/50 사전학습 + fine-tune |
