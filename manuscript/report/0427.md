## 1. 배경 및 목표

### 문제 정의

네트워크 침입 탐지(IDS) 데이터셋(CIC-IDS2017/2018, UNSW-NB15)에서 **tail 클래스 F1을 단일 XGBoost 베이스라인보다 높이는 것**이 핵심 목표다. Imbalance Ratio(IR)가 1,000을 초과하는 tail 공격 클래스(Heartbleed: train 7개, Infiltration: train 22개 등)에서 XGBoost 베이스라인의 recall이 붕괴한다.

### 직전까지의 실패 요약

| 스크립트 | 아이디어 | 결과 |
|---|---|---|
| `cic_6.py` | Taxonomy-aware MoE (TAILGUARD) | 베이스라인 대비 개선 없음 |
| `cic_xg_19.py` | XGBoost 라우터 + 전문가별 SMOTE | 베이스라인 대비 개선 없음 |
| `code_final.py` | OOD 필터 + 역량 가중 앙상블 | 개선 없음 |
| `code_chrono.py` / `code_file.py` | 시간/파일 기반 분할(leakage 방지) | 성능 하락 |
| `code_mat.py` | FAR-MoE: CM 유도 전문가 + OOD 게이팅 | val 거의 완벽, test 개선 없음 |

핵심 실패 원인 네 가지:
1. **라우팅 오류가 불균형을 증폭**: tail 클래스가 드물어 라우터가 학습 못 함 → 잘못된 전문가로 라우팅 → 베이스라인보다 악화
2. **전문가 내 불균형 미해결**: 클래스를 전문가에 분배해도 각 전문가의 IR은 그대로
3. **혼동 행렬 leakage**: val 분포의 CM으로 설계한 전문가가 test 분포에 일반화 안 됨
4. **XGBoost의 내재적 MoE**: 부스팅이 이미 hard sample을 반복 가중 → 명시적 라우팅이 추가 이득 없음

## 2. 이번 세션에서 수행한 작업 (2026-04-24)

### 2.1 새로운 방향: Synthetic Benchmark로 전환

**핵심 가설**: CIC-IDS에서 MoE가 실패하는 것이  
(a) **도메인 특화 데이터 아티팩트** 때문인지, 아니면  
(b) **혼동행렬 기반 전문가 설계 자체가 근본적으로 취약**한 것인지  
를 구분하기 위해, 잘 연구된 균형/불균형 benchmark(CIFAR-10/100)에서 동일한 구조를 테스트하는 방향으로 전환했다.

두 가지 신규 스크립트가 구현되었다.

### 2.2 `code_dce.py` — DCE-XGB (Dual-Balance Collaborative Experts)

#### 구조

ICML 2025 DCE 논문 아이디어를 XGBoost로 구현:

```
세 XGBoost 전문가 (동일 훈련 데이터, 다른 sample weighting):
  head     : log-inverse frequency (다수 클래스 가중치 유지)
  balanced : inverse frequency (표준 균형 가중)
  tail     : power-law upweighting (희귀 클래스 공격적 가중)

Dynamic Expert Selector (DES):
  1. val에서 전문가별 per-class recall → affinity matrix 계산
  2. 각 클래스를 최적 전문가에 배정
  3. 클래스별 Gaussian pseudo-sample 생성 (centroid/covariance 기반)
  4. LogisticRegression 라우터를 pseudo-sample로 훈련
  5. 추론: α = router.predict_proba(x), 최종 확률 = Σ_e α_e · proba_e(x)
```

#### 하이퍼파라미터

| 파라미터 | 값 |
|---|---|
| n_estimators | 200 |
| max_depth | 6 |
| learning_rate | 0.1 |
| subsample / colsample_bytree | 0.8 / 0.8 |
| tail_power | 2.0 |
| n_pseudo_per_class | 300 |
| PCA 차원 (CIFAR-100) | 256 |
| seed | 42 |

#### 실험 1: CIFAR-10LT (Long-Tail, IR=100)

단순 7-클래스 long-tail 설정 (class_0 = 1000, class_6 = 10개):

| 지표 | Baseline | DCE-XGB | Δ |
|---|---|---|---|
| macro-F1 | 0.1718 | 0.1938 | **+0.0220** |

Per-class 결과:

| 클래스 | 지지도 | f1(B) | f1(DCE) | Δ |
|---|---|---|---|---|
| class_0 | 1000 | 0.3607 | 0.3999 | +0.0392 |
| class_1 | 1000 | 0.4459 | 0.4560 | +0.0101 |
| class_4 | 1000 | 0.1801 | 0.2552 | +0.0751 |
| class_5 | 1000 | 0.0946 | 0.1258 | +0.0312 |
| class_6 | 1000 | 0.0218 | 0.0502 | +0.0285 |

> 모든 클래스에서 개선. tail class(class_6)도 +0.0285 향상.

#### 실험 2: CIFAR-100Tasks (CIFAR-100 superclass 구조, 20 tasks)

**데이터셋 구성**:
- 20개 superclass 태스크 (각 superclass당 5개 fine-grained class)
- 균형 태스크(IR=1): 10개 superclass (aquatic_mammals, fish, 등)
- IR=50 태스크: 5개 superclass (flowers, fruit_and_vegetables, 등)
- IR=100 태스크: 5개 superclass (insects, people, household_furniture, 등)
- 전체 100개 클래스, train=26,088 / val=6,522 / test=10,000

**결과** (최종 실행: `dce_20260424_073622`):

| 지표 | Baseline | DCE-XGB | Δ |
|---|---|---|---|
| macro-F1 | 0.1459 | 0.1525 | **+0.0066** |
| weighted-F1 | 0.1459 | 0.1525 | **+0.0066** |

주목할 클래스별 개선:
- `turtle`: 0.0829 → 0.1641 (+0.0812)
- `television`: 0.1818 → 0.2544 (+0.0726)
- `chimpanzee`: 0.2719 → 0.3269 (+0.0550)
- `willow_tree`: 0.1533 → 0.2079 (+0.0546)

일부 클래스 하락:
- `sunflower`: 0.1441 → 0.0755 (−0.0687)
- `possum`: 0.0855 → 0.0265 (−0.0590)
- `plate`: 0.4305 → 0.3755 (−0.0549)

**이전 실험 비교** (CIFAR-100Tasks 설정별):

| 실험 | tasks 구성 | Baseline macro-F1 | DCE macro-F1 | Δ |
|---|---|---|---|---|
| `dce_072946` | 10 tasks (5 balanced, 2 IR=50, 3 IR=100) | 0.1514 | 0.1614 | +0.0099 |

**라우터 affinity 패턴**: 대부분의 클래스에서 `balanced` 전문가가 최적으로 선택됨. 극도로 희귀한 클래스(train 4개 미만)는 recall=0 으로 DCE도 개선 불가.

### 2.3 `code_3.py` — ConfMat-MoE (CIFAR benchmark)

#### 구조

`code_mat.py`의 혼동행렬 기반 전문가 설계를 CIC-IDS가 아닌 **CIFAR-10/100**에 적용하여, 실패 원인이 도메인 아티팩트인지 설계 자체의 문제인지 검증:

```
Phase 1 (PROBE): train 60%에서 XGBoost 훈련 → val 20%에서 예측 → 정규화 혼동 행렬
Phase 2 (CLUSTER): off-diagonal conf[i,j] > τ 인 클래스 쌍을 union-find로 클러스터링
Phase 3 (EXPERT): K+1 XGBoost 전문가 훈련
  - Anchor 전문가: 균일 가중
  - ConfMat 전문가_k: 클러스터 k 내 클래스에 w_focus(=5) 가중
Phase 4 (ROUTER): 각 전문가에 Logistic Regression 역량 모델 훈련 (val에서 정답 여부 레이블)
추론: final_proba = Σ_e α_e(x) · proba_e(x)
```

#### 결과 — CIFAR-10 (균형 데이터셋, 10 클래스)

| 지표 | Baseline | ConfMat-MoE | Δ |
|---|---|---|---|
| macro-F1 | 0.4992 | 0.5066 | **+0.0074** |

Phase 1에서 10개 클래스가 **단일 클러스터**로 묶임 (모든 클래스가 서로 혼동됨). 따라서 ConfMat 전문가는 사실상 모든 클래스에 w_focus를 부여하는 전문가가 됨.

Per-class 결과:
| 클래스 | f1(B) | f1(MoE) | Δ |
|---|---|---|---|
| airplane | 0.5681 | 0.5812 | +0.0131 |
| cat | 0.3476 | 0.3714 | +0.0237 |
| dog | 0.4022 | 0.4152 | +0.0130 |
| truck | 0.5604 | 0.5616 | +0.0011 |

모든 클래스에서 소폭 개선.

#### 결과 — CIFAR-100 (100 클래스)

Phase 1에서 **5개 혼동 클러스터** 발견:
- Cluster 0: 11개 클래스 (야외/자연 물체 계열)
- Cluster 1: 62개 클래스 (최대 규모, 대부분 동물/사물)
- Cluster 2: 3개 클래스
- Cluster 3, 4: 각 2개 클래스

> 실험이 훈련 중 타임아웃 또는 인터럽트로 최종 test 결과가 로그에 기록되지 않음.

## 3. 핵심 관찰 및 시사점

### 3.1 DCE-XGB: 첫 번째 양의 결과

- **CIFAR-10LT에서 macro-F1 +0.0220**: 모든 클래스에서 개선. tail 클래스(class_6, IR=100)도 개선.
- **CIFAR-100Tasks에서 macro-F1 +0.0066**: 소폭 개선. IR=100 tail 클래스 중 일부는 크게 개선(turtle: +0.0812), 일부는 하락.
- **CIC-IDS 대비 성공 이유**: CIFAR은 각 클래스당 test sample이 충분(각 100개)하고, 분포 이동이 없는 i.i.d. 환경. CIC-IDS처럼 시간적 leakage나 file-level 분포 이동이 없음.

### 3.2 ConfMat-MoE: CIFAR-10에서 소폭 개선

- CIFAR-10에서 +0.0074 개선은, `code_mat.py`의 실패가 **CM leakage + CIC-IDS 분포 이동** 때문이었음을 시사.
- 단, CIFAR-10에서 모든 클래스가 하나의 클러스터로 묶인 것은 clustering threshold τ=0.05가 너무 낮다는 의미. 전문가가 실질적으로 분화되지 않음.

### 3.3 DCE affinity 패턴 — `balanced` 전문가의 지배

CIFAR-100Tasks에서 대부분의 클래스가 `balanced` 전문가를 최적으로 선택. 이는:
- `tail` 전문가의 power-law 가중이 지나치게 희귀 클래스에 집중되어 일반 클래스의 분류 품질을 희생시키는 trade-off가 있음을 시사.
- 혹은 CIFAR 특성상 IR이 CIC-IDS만큼 극단적이지 않아(max IR=100 vs CIC IR=1000+) `tail` 전문가의 이점이 제한적임.

## 5. 다음 방향 (Open Questions)

1. **CIC-IDS2017에 DCE-XGB 적용**: CIFAR에서 긍정적 결과를 얻었으므로, CIC-IDS2017에서 DCE-XGB를 직접 실행. 단, `subsample_train` 파라미터로 500,000개 이하로 서브샘플링 필요.

2. **CIFAR-100 ConfMat-MoE 완료**: code_3.py의 CIFAR-100 실험이 중단됨. 완료하여 혼동 클러스터 5개 전문가 설계의 test F1 확인 필요.

3. **tail_power 튜닝**: DCE에서 `balanced` 전문가가 대부분의 클래스를 가져가는 현상 → `tail_power`를 낮추거나 가중치 clip 상한을 조정하여 전문가 분화를 촉진.

4. **비대칭 앙상블**: IR=100 이상 클래스에는 tail 전문가의 출력을 더 많이 반영하는 클래스별 혼합 가중 전략 검토.
